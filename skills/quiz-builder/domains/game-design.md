# Domain Pack: Game Design

Scenarios for a game-design course. The student reasons about mechanics, player experience, and balance, and separates the parts of a design.

## Scenario entities and decisions

- A designer adjusting a mechanic and predicting how player behavior and feel will change.
- A game with a pacing, difficulty, or motivation problem to diagnose.
- A choice among player types or motivations a feature is meant to serve.
- A systems decision about feedback loops, rewards, or progression.

## Metrics, frameworks, and vocabulary (with realistic ranges)

- Frameworks: the MDA framework (mechanics, dynamics, aesthetics), the core loop, flow (matching challenge to skill), Bartle's player types (achiever, explorer, socializer, killer), intrinsic versus extrinsic motivation, positive and negative feedback loops, progression and pacing, onboarding, emergence versus scripted design, game feel and juice.
- Vocabulary: difficulty curve, reward schedule, balance, mastery, agency, the magic circle, retention and session length where the course treats them.

## What good distractors look like here

- Confusing mechanics (the rules the designer sets) with dynamics (the behavior that emerges when players act on them).
- Treating grind or time spent as engagement, rather than meaningful challenge or motivation.
- Confusing difficulty (raw hardness) with challenge (difficulty matched to player skill, the flow idea).
- Missing how a positive feedback loop snowballs a lead and ends matches early, where a negative loop keeps them close.
- Assuming better graphics or more content equals more fun.

## House style: two worked examples

Scenario, hard: "A designer buffs the leader in a racing game so the player in first pulls further ahead each lap. Players report matches feel decided early. Which concept explains this, and what is a fix?" Correct answer: a positive feedback loop is amplifying the lead; a negative feedback loop (rubber-banding, catch-up items) would keep races competitive. Distractors: the difficulty is too high, the core loop is broken, players need more content.

Scenario, medium: "Using MDA, a player says the game feels 'tense and exciting.' Which layer is that, and where does the designer act to change it?" Correct answer: tension is an aesthetic (the felt experience); the designer cannot set it directly and instead tunes mechanics, which produce the dynamics that create the aesthetic. Distractors: it is a mechanic to set directly, it is a dynamic the designer writes, it is unrelated to MDA.

## Feedback voice

Correct-answer feedback names the layer or concept (which MDA layer, which feedback loop, the flow idea). Incorrect-answer feedback names the confusion, for example mechanics versus dynamics or difficulty versus challenge, without restating the answer.
